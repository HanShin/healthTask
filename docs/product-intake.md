# 운동 기록 앱 요구사항 정리 (초안)

## 1) 현재까지 확인된 요구사항

1. 기본적인 운동 기록
   - 세트 수, 반복 횟수, 무게 기록
2. 유산소 운동 기록
   - 유산소 운동 종류와 수행 시간/강도 기록
3. 캘린더 기록 조회
   - 날짜별 운동 내역(종목/볼륨/시간) 확인
4. 주간/월간 통계
   - 주 단위, 월 단위 운동량 집계 확인
5. 루틴 관리
   - 루틴 작성, 수정, 보관
6. 운동 종목 안내 + 유튜브 연동
   - 운동명 기반으로 자세 참고를 위한 유튜브 빠른 이동
7. 자극 부위 정리
   - 각 운동별 타겟 근육(주/보조) 정보 제공

## 2) 제안 기술 스택 (요청사항 반영)

- Backend: Java (Spring Boot 권장)
- View Template: Handlebars (SSR)
- DB: SQLite (개인용/저용량 요구에 적합)
- 배포 형태: 모바일 Safari(iPhone 17 Pro Max) 기준 반응형 웹앱(PWA 옵션)

## 3) 추가로 확정이 필요한 항목

### A. 계정/보안
- 회원가입 없이 로컬 단일 사용자로 사용할지
- 간단 비밀번호(앱 잠금) 필요 여부

### B. 기록 단위
- 무게 단위: kg만 쓸지, lb도 지원할지
- 유산소 단위: 시간(분), 거리(km), 칼로리 중 무엇을 입력/표시할지
- 휴식 시간(세트 간 rest) 기록 필요 여부

### C. 캘린더/통계 기준
- 주간 기준: 월~일 vs 일~토
- 월간 통계 항목: 총 운동일수, 총 볼륨(세트×반복×무게), 총 유산소 시간 등
- 목표(예: 주 4회) 대비 달성률 표시 필요 여부

### D. 루틴 기능 범위
- 루틴 템플릿 복제 기능 필요 여부
- 루틴 순서/슈퍼세트/드롭세트 등 고급 기능 필요 여부
- 운동 완료 체크(체크리스트) UX 필요 여부

### E. 운동 라이브러리/유튜브
- 초기 기본 운동 종목 개수(예: 50개/100개)
- 유튜브 연결 방식
  - 고정 추천 영상 링크
  - 운동명 검색 결과 페이지로 이동
- 한글/영문 운동명 동시 지원 여부

### F. iPhone 최적화
- 하단 탭 네비게이션 선호 여부
- 다크모드 지원 필요 여부
- 오프라인 저장(PWA) 필요 여부

## 4) 권장 MVP 범위

1. 운동 기록(근력/유산소)
2. 캘린더 조회
3. 주간/월간 기본 통계
4. 루틴 CRUD
5. 운동 라이브러리 + 유튜브 검색 링크
